# 人工智能基础综合实践实验报告草稿

## 基本信息

- 是否组队：是
- 组名：每日 AI 与机器人技术情报数字员工系统
- 选择框架：OpenClaw 风格多数字员工协作设计
- 选题：每日 AI 与机器人技术情报分析

## 成员分工

| 成员 | 负责模块 | 具体任务 |
| --- | --- | --- |
| 队友 1 | AI 技术员工 | 跟踪大模型、Agent、多模态、开源项目，分析技术新颖性、学习价值和工程复现价值 |
| 组长 | 机器人与具身智能员工 | 跟踪 ROS2、SLAM、导航、机器人操作，分析工程落地条件和复现实验价值；同时负责服务器部署、网页运行、代码整合和总报告组织 |
| 队友 2 | 产业与产品员工 | 跟踪机器人公司、AI 产品、商业化场景、投融资动态，分析产品趋势和真实应用风险 |
| 总控员工 | 报告整合 | 汇总三个员工输出，生成每日技术日报 |

## 项目整体思路

本项目没有采用作业示例中的股票市场分析，而是选择更贴合人工智能课程的“AI + 机器人技术情报”主题。系统每天收集 AI、机器人和产业产品相关公开信息，再交给三个数字员工分工分析，最后由总控员工生成一份结构化 Markdown 日报。

项目希望解决的问题是：人工查找技术动态耗时较长，单个大模型总结又容易结构混乱、来源不清。因此，本项目采用多员工协作方式，把技术、工程、产业三个视角拆开处理，并在最终报告中保留来源链接、价值判断、风险提醒和推荐等级。

## 数据来源与分析指标

数据来源包括：

- demo 数据：用于课堂演示，保证没有网络和 API Key 时也能运行。
- RSS：用于采集科技媒体和机器人行业动态。
- arXiv：用于采集 AI、机器人、机器学习方向论文。
- GitHub：用于采集开源项目和工程趋势。

分析指标包括：

1. 技术新颖性：是否出现新模型、新方法、新工具或新基准。
2. 工程价值：是否有复现、部署、集成或课程项目实践价值。
3. 学习价值：是否适合作为后续学习主题。
4. 商业价值：是否与真实产品、公司、产业场景有关。
5. 难度评估：是否涉及复杂模型、机器人系统或仿真到现实迁移。
6. 推荐等级：根据综合评分给出 A、B、C 级关注建议。

为避免 AI 编造，系统只基于已采集到的标题、摘要、正文片段和来源链接进行分析。报告附录会保留每条信息的来源，答辩时可以追溯。

## 技术实现方案

后端技术：

- Python
- Streamlit
- Requests
- feedparser
- Pydantic
- PyYAML

AI 模型：

- 默认使用 mock 模式，保证课堂演示稳定。
- 可配置 OpenAI、SiliconFlow、Anthropic 等外部 API。
- 2C2G 云服务器不本地运行大模型，只调用外部 API，降低服务器压力。

数据处理：

- 采集模块负责读取 demo、RSS、arXiv、GitHub。
- 分类模块负责把信息分为 AI、机器人、产业三类。
- 评分模块负责给出新颖性、工程价值、学习价值、商业价值和难度。
- 报告模块负责生成 Markdown 日报。

前端技术：

- Streamlit 公网 UI。
- 页面包含项目总览、日报预览、数据与过程、答辩材料四个部分。

部署技术：

- Linux 云服务器。
- Streamlit 监听 `0.0.0.0:8501`。
- 通过公网 IP 访问 UI。
- 后续可扩展为 systemd 后台常驻服务。

## 实验与方案对比

| 方案 | 做法 | 优点 | 不足 |
| --- | --- | --- | --- |
| 纯人工整理日报 | 人工搜索资料并总结 | 可控、成本低 | 耗时长，难以每天稳定更新 |
| 单个大模型一次性总结 | 把全部信息交给一个模型生成报告 | 实现简单 | 分工不清，来源追溯弱，容易遗漏 |
| 三数字员工协作 | 三个员工分别分析，总控员工整合 | 分工明确，结构清晰，便于答辩展示 | 需要设计流程、配置和 UI |

最终选择第三种方案，因为它更符合“组队协作数字员工”的作业要求，也能更清楚地体现每位成员的贡献。

## 云服务器部署

服务器资源为 2 核 2G，并且已经运行 DDNet 服务器。因此本项目采用轻量部署方案：

1. 不在服务器本地运行大模型。
2. 只部署 Python + Streamlit 项目。
3. 大模型分析通过外部 API 完成。
4. demo 模式可在无 API Key 情况下稳定演示。
5. Streamlit 使用 8501 端口提供公网访问。

启动命令：

```bash
streamlit run app.py --server.address 0.0.0.0 --server.port 8501
```

公网访问地址：

```text
http://47.97.114.62:8501
```

## 项目创新点

1. 从财经分析改为 AI + 机器人技术情报分析，更贴合人工智能课程。
2. 三个员工分别覆盖技术、工程、产业三个视角，不是简单重复总结。
3. 报告不只输出新闻摘要，还包含学习建议、工程价值、风险提醒和推荐等级。
4. 系统保留来源链接，降低 AI 编造风险，方便答辩追溯。
5. 支持 demo 模式和真实数据模式，兼顾课堂稳定演示和真实部署扩展。
6. 用 OpenClaw Skill 化思路拆分采集、摘要、评分、报告生成能力。

## 遇到的问题与解决方法

1. 云服务器资源有限  
   服务器只有 2 核 2G，且已有 DDNet 服务运行。解决方法是不在本地跑大模型，只调用外部 API，并限制每位员工分析条数。

2. 真实 API 成本和 Token 控制  
   如果每条信息都调用大模型，成本会增加。解决方法是默认使用 mock 或只让总控员工调用真实模型，同时限制采集条数。

3. 网络数据源不稳定  
   RSS、arXiv、GitHub 可能出现访问失败。解决方法是提供 demo 模式和失败回退机制，保证课堂演示不受影响。

4. AI 可能编造内容  
   解决方法是保留每条来源链接，并在提示词中要求模型不能编造来源中没有的公司、数字或结论。

5. 作业模板重复风险  
   解决方法是不用股票分析结构，而是围绕 AI 技术学习和机器人方向设计自己的员工分工、指标和创新点。

## 实验总结

本项目实现了一个每日 AI 与机器人技术情报数字员工系统。系统通过三个数字员工完成不同方向的分析，再由总控员工生成综合日报。项目既能在 demo 模式下稳定演示，也能扩展到真实数据采集和外部大模型 API。

通过本项目，我们完成了数据采集、分类、评分、报告生成、Web UI 和云服务器部署流程，体现了多智能体协作系统在技术情报分析场景中的应用价值。
