# 项目技术实现说明

本项目根据上级目录中的 `digital_employee_tech_spec.md` 实现，并结合综合大作业要求补充了真实数据采集、真实 LLM Agent 调用、公网 UI 展示和原创分工说明。项目主题不是股票分析模板，而是“每日 AI 与机器人技术情报数字员工系统”。

## 运行闭环

```text
读取配置
  -> demo 数据或真实数据采集
  -> 信息分类
  -> 三个数字员工分别分析
  -> 总控员工生成 Markdown 日报
  -> 保存 reports/daily_report.md
  -> Streamlit 页面展示与下载
```

## 数字员工

- AI 技术员工：由队友 1 负责，关注大模型、Agent、多模态和开源项目，输出技术新颖性、学习价值和工程复现价值判断。
- 机器人与具身智能员工：由组长负责，关注 ROS2、SLAM、导航和机器人操作，输出工程落地条件、复现实验价值和风险提醒；组长同时负责服务器部署、网页运行和总报告组织。
- 产业与产品员工：由队友 2 负责，关注机器人公司、AI 产品、商业化场景和投融资动态，输出产品趋势、市场价值和真实应用风险。
- 总控员工：整合三个员工结果，生成“技术学习 + 工程实践 + 产品趋势”综合 Markdown 报告。

## 作业要求对应关系

| 作业要求 | 本项目对应实现 |
| --- | --- |
| 三人一组，每人对应一个员工 | AI 技术员工、机器人与具身智能员工、产业与产品员工 |
| 每日分析主题 | 每日 AI 与机器人技术情报分析 |
| 拆解任务并整合总报告 | 三员工分别分析，总控员工合成日报 |
| UI 程序展示 | Streamlit 公网页面展示流程、结果和创新点 |
| 云服务器部署 | 2C2G Linux 服务器运行 Streamlit，调用外部 API，避免本地大模型占用资源 |
| 避免 AI 编造 | 保留来源链接，报告附录列出信息来源 |

## 方案对比

| 方案 | 特点 | 选择结论 |
| --- | --- | --- |
| 纯人工整理日报 | 成本低但效率低，难以每天稳定更新 | 作为对照方案 |
| 单个大模型总结 | 实现简单，但分工不清，答辩时难体现组队协作 | 不作为最终方案 |
| 三数字员工协作 | 分工清楚，可解释性强，能保留来源和评分 | 最终采用 |

## 原创性说明

本项目从课程中的多智能体协作要求出发，选择 AI 技术与机器人方向作为每日分析主题。系统不是照搬股票分析示例，而是从技术学习、工程复现和产业产品三个角度形成互补分析。报告中包含学习建议、工程价值、风险提醒和推荐等级，便于在课程答辩中展示“信息获取 -> 多员工分析 -> 总控汇总 -> UI 展示”的完整闭环。

## 配置

- `config.yaml`：demo/mock 模式，保证课堂演示时不依赖网络、API Key 或第三方服务。
- `config.real.yaml`：真实数据模式，从 RSS、arXiv、GitHub 采集公开信息，并支持 OpenAI/Anthropic 真实 LLM Agent 分析。

## 真实部署边界

真实数据模式依赖运行环境能访问 RSS、arXiv 和 GitHub。真实 LLM 未配置 API Key 时，系统会回退到 mock 分析，保证部署环境下流水线仍能跑完。
